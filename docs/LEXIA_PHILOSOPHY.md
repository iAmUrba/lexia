# FILOSOFÍA Y GOBERNANZA DE LEXIA

> *"LexIA no existe para que el usuario aprenda un sistema. Existe para que el sistema aprenda cómo trabaja el usuario."*

## Principios Fundamentales
- **LexIA no tiene pantallas.** Tiene capacidades dentro del Sistema Operativo.
- **La IA es invisible.** Es electricidad por debajo del sistema; jamás un módulo o una pantalla que diga "Bienvenido a la IA".
- **El Sidebar es patrimonio histórico.** Mover un botón cinco centímetros es perder productividad del despacho.
- **El usuario nunca piensa.** Todo módulo debe responder automáticamente tres preguntas: *¿Qué estoy viendo? ¿Qué puedo hacer? ¿Qué sigue?*
- **El tiempo del funcionario vale más que el código.**
- **Ningún módulo conoce a otro módulo.** Se comunican por eventos, garantizando un desacoplamiento brutal.

## Checklist Obligatorio (Antes de cualquier Pull Request)
1. [ ] ¿Ahorra al menos 15 minutos por semana a un funcionario?
2. [ ] ¿Respeta el Design System y el Application Shell?
3. [ ] ¿Puede implementarse usando los Componentes Base (UI/Layout) sin crear CSS nuevo?
4. [ ] ¿Respeta la compatibilidad hacia atrás?
5. [ ] ¿Está a máximo 3 clics de distancia?
6. [ ] ¿El usuario entendería esta función sin capacitación?
7. [ ] ¿Se puede desinstalar sin afectar a los demás módulos?
8. [ ] ¿La IA permanece invisible?
9. [ ] ¿Hace que LexIA parezca más inteligente y no más complicada?
